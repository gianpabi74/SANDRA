# ADR-0017 — Execution Use Case

## Stato

Accepted and immutable.

Candidate generated by the staged R3-000017A sequence, independently frozen
by R3-000017B-1 and published by R3-000017B-2.

## Decisione candidata

SANDRA uses the existing canonical capability `execution` and consumes the
canonical `ExecutionPlan` produced by Planning.

The Application vertical consists of:

- `ExecutionRequest`;
- `ExecutionStepOutcome`;
- `ExecutionResult`;
- `ExecutePlanPort`;
- `ExecutionEngine`;
- `ExecutePlan`.

Execution owns:

- execution lifecycle;
- idempotency;
- bounded retry;
- partial results;
- recovery invocation.

Execution excludes:

- policy decision;
- plan creation;
- verification conclusion.

A completed command is recorded as an execution outcome, not as proof that the
Habitat reached the desired postcondition. Verification remains a separate
capability and receives resulting evidence and partial outcomes.
