# ADR-0015 — Policy Decision Use Case

## Stato

Accepted and immutable.

Candidate generated by R3-000015A, verified against its exact exported
manifest and published by R3-000015B.

## Decisione candidata

SANDRA uses the existing canonical capability identifier:

`policy_decision`

The capability evaluates authority, risk, limits and conditions for one
proposed action.

The canonical outcomes are:

- allow;
- deny;
- conditional.

Every decision contains:

- request and action correlation;
- approval requirement;
- explicit limits;
- one or more reasons;
- decision and expiry timestamps.

## Inputs

The request references:

- one managed subject;
- one proposed action;
- one Desired State;
- one or more qualified evidence records;
- one Resource Graph snapshot;
- immutable evaluation context;
- immutable requested limits.

## Boundaries

Policy Decision owns:

- allow;
- deny;
- conditional outcome;
- approval requirement;
- limits;
- reasons;
- decision expiry.

Policy Decision excludes:

- execution;
- transport;
- verification.

The use case does not:

- observe the Habitat;
- qualify evidence;
- mutate Desired State;
- create an execution plan;
- invoke an adapter;
- execute an action;
- conclude postcondition verification.
