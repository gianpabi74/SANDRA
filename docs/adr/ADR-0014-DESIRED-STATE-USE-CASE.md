# ADR-0014 — Desired State Use Case

## Stato

Accepted and immutable.

Candidate generated by R3-000014A, verified against its exact exported
manifest and published by R3-000014B.

## Decisione candidata

SANDRA represents approved intent through immutable, generation-controlled
Desired State records.

`DeclareDesiredState` receives a `DesiredStateDeclaration` and persists a new
`DesiredStateRecord` through `DesiredStateRepository`.

Each declaration contains:

- the managed subject reference;
- an approval reference;
- expected generation;
- desired configuration;
- desired service state;
- declared limits.

## Concurrency

Generation zero represents absence of a previously declared Desired State.

Every successful declaration increments the generation by exactly one.

The repository must enforce optimistic concurrency when saving the new
generation.

## Boundaries

Desired State owns:

- desired configuration;
- desired service state;
- declared limits;
- intent generation.

Desired State excludes:

- live telemetry;
- imperative commands;
- adapter configuration.

The use case does not:

- observe live state;
- qualify evidence;
- evaluate policy;
- create execution plans;
- execute actions.
