# RV-000001 — Regola Inviolabile

## Titolo

Nessun Runbook senza percorso deterministico verso STATUS=PASS.

## Regola

Un Runbook non può essere progettato, consegnato o considerato
completato se non esiste un percorso oggettivo, verificato e
deterministico che porti a STATUS=PASS.

## Principi

1. Nessuna intuizione o supposizione può diventare codice operativo.
2. Ogni comando deve essere:
   - verificato nell'ambiente reale; oppure
   - supportato dalla documentazione ufficiale e coerente con
     l'ambiente verificato.
3. Tutti i prerequisiti devono essere verificati prima della modifica.
4. Un FAIL del Runbook è un bug del Runbook.
5. Se manca una verifica oggettiva si esegue prima un Preflight.
6. Ogni Runbook introduce una sola capability certificabile.
7. Solo i Runbook PASS entrano nella baseline di SANDRA.
